Plantbox

A self-initiated app concept that lets travelers safely leave their plants in a plant hotel while away.

Team: Group project (2 members)

Course:Intarction design

Focus: Interaction design, mobile UI

Tools: Figma

Project overview

Plantbox is a mobile app concept designed to solve a simple but common problem: what to do with houseplants when traveling. The idea of a plant hotel was developed by us and does not currently exist as a real service. The goal was to design an intuitive booking flow and a calm, trustworthy interface.

The concept

Users can book a stay for their plants by selecting travel dates, adding plant details such as type, size and care needs, and completing a reservation. Optional pickup and delivery services can be added before checkout. After booking, users receive a room number and digital access tools to manage their stay.

Interaction flow

The interaction design focuses on guiding users step-by-step through a clear booking journey. Each screen presents one primary task at a time, reducing cognitive load and making the process feel simple and predictable.

  • Select dates in calendar
  • Add plants with type, size and care requirements
  • Log in or create account
  • Review and confirm booking
  • Receive room number and access tools

Visual system & UI design

The UI was designed to feel calm and trustworthy, using soft colors, rounded components and clear typography. Consistent buttons and navigation patterns support predictable interaction, while generous spacing keeps the interface easy to read on small screens.

Key features

After completing a booking, users are shown a confirmation screen with a room number and four primary actions.

  • View QR code for hotel and room access
  • View map of plant hotel rooms
  • Get directions to the hotel
  • Contact the plant hotel

A bottom navigation menu provides access to the QR code, user profile and calendar. If no bookings exist, users can start a new reservation directly from the calendar view.

Design decisions for usability

The booking flow was structured as a linear journey with clear progress, helping users understand what to do next at all times. Persistent bottom navigation supports fast access to important functions without interrupting the main flow.
