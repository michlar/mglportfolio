Event Planning Web Application for Accenture

Improving clarity and flow in internal event scheduling interfaces.

Role: UI Designer

Client: Accenture (Attract)

Timeline: Spring 2023

Focus: Interface structure, visual hierarchy, interaction

Project overview

Accenture’s internal event planning process was spread across multiple tools and channels. This created fragmented workflows, unclear structure and unnecessary friction for employees planning and following up events.

The goal was to design a unified web application that brings planning, tracking and follow-up into one calm and structured interface.

The challenge

Employees needed an interface that made it easy to plan new events, track tasks and progress, find past events and history, and follow up consistently across different school groups.

  • Reduce fragmentation across tools and pages
  • Make status and progress easy to understand at a glance
  • Support a consistent flow for planning and follow-up

Interface exploration

I started by exploring layout systems and navigation patterns that could reduce the number of clicks needed to complete key tasks. Early sketches tested how dashboards, calendars and event lists could be structured to give immediate overview without overwhelming the user.

Refinements from feedback

Feedback from internal users revealed where visual cues and hierarchy were unclear. I refined status indicators, list structure and icon placement to make actions and system state easier to understand at a glance.

Visual system & UI design

The high-fidelity design builds on Accenture’s design guidelines, allowing the work to focus on layout, rhythm, spacing and interaction patterns rather than reinventing visual style.

Consistent components and predictable navigation create a calm interface across dashboards, calendars and event flows.

Final interface

The final solution brings planning, task tracking and follow-up into one unified interface. Dashboards provide quick shortcuts to key actions, event lists support overview and filtering, and creation flows guide users through required inputs with clear structure.

What I learned

  • How to shape calm interface rhythm for complex internal tools
  • How to make dense information easier to scan through hierarchy and spacing
  • How small UI adjustments can significantly improve clarity and confidence

If I continued the project, I would explore micro-interactions to guide attention in dense views and make state changes even more obvious.
